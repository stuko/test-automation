* 설치할 서버에서, 최초 다운로드 : test-automation 폴더 다운로드
  - 설치된 IP : kanboard 설치 IP = test-controller IP = Mongo DB IP

* 다운로드 폴더(test-automation)로 이동
  : 먼저 kanboard를 설치 하고 실행 해야 함.
  - cd ./kanboard/
  - sudo docker-compose -f docker-compose-with-mariadb.yml down
  - sudo docker-compose -f docker-compose-with-mariadb.yml up -d
  - cd ../

* kanboard 의 webhooking 설정 과 API 토큰정보를 구함.
  - 계정의 설정 메뉴 / Webhook / Webhook URL : http://{test-controller}:5000/controller 저장
  - 계정의 설정 메뉴 / API / API 토큰 정보를 /test-automation/volume/config/config.json 파일의 kanboard_token 항목에 저장


1. http://{kanboard 설치 IP}:{8080:디폴트}/ 로 접속 (최초 admin/admin) : 성공 확인

2. /test-automation/install_all.sh 실행
  - http://{test-controller IP}:5000/testing 로 접속 (test-controller 정상 확인)

3. MongoDB 접속 (클라이언트 설치 후) : {MongoDB IP:설치된서버 IP}:{27017:디폴트} , 계정/패스워드 없음.

4. 프로젝트 생성
  - kanboard 에 접속하여, 작업에 사용할 프로젝트를 생성함.
  - 정상적으로 Webhooking이 설정 되었으면, 요구사항을 등록시, Kanboard 구조가 재구성 됨
    : 프러덕트 백로그-스프린트백로그-개발-테스트자동화-테스트리뷰-테스트완료-릴리즈

5. 최초 시스템 설정
  - JMeter 플러그인을 사용하여 연결 작업을 해줌 : kanboard <--> test-controller <--> 젠킨스,메터모스트
  - JMeter 는 test-automation을 다운로드 하면, 포함되어 있음(플러그인과 필요 라이브러리가 포함됨)
    * jmeter 실행시 test-controller 가 실행 중인 URL을 입력해 줘야 함.
    * jmeter -DTEST_URL=http://{test-controller IP}:5000/ 실행
    * File / Open / Kai-s-scenario.jmx 파일을 선택합니다.(Kai-S 테스트 시나리오 있음)
    * Thread Group을 선택하고 , Test Automation Plan Main 을 클릭합니다.
    * "프로젝트 연결 설정" 탭에서 앞서 생성한 "프로젝트" 가 보이면, 선택합니다.
    * Jenkins 서버 URL, Jenkins프로젝트명, Jenkins 토큰, 메타모스트 후킹 키 값을 입력하고, "테스트 자동화 연결" 저장을 클릭합니다.

6. 테스트 시나리오 작성
  - Jmeter를 실행 합니다. (jmeter -DTEST_URL=http://{test-controller IP}:5000/)
  - 만들고자 하는 Plan 파일을 생성합니다.
  - Thread Group 을 추가 합니다.
  - Add / Sampler / "테스트 자동화 플러그인" 을 선택합니다.
  - Jmeter를 활용한 시나리오 작성방법은 별도의 Jmeter 자료를 활용하시면 됩니다.
  - 테스트 데이터를 생성합니다.
    : Test Automation Plan / 테스트 데이터 설정 탭을 선택하고, 테스트 데이터 형식을 입력하고 "테스트 데이터 설정 저장" 을 클릭합니다.
  - 테스트 실행 설정을 저장 합니다.
    : 생성된 테스트 데이터를 별도의 다른 서버로(TCP/Kafka) 보내야 할 경우, TCP 혹은 Kafka를 선택하고, 설정 정보를 입력합니다.
    : 만약 Jmeter의 시나리오만 실행 하려면, DEFAULT로 선택하고 "테스트 실행 정보 저장"을 클릭합니다.

7. kanboard , jenkins, jmeter를 활용하여 테스트를 즐기시기 바랍니다.
